# Security Policy

## Supported versions

Security fixes are provided for the latest version on the `main` branch.

## Reporting a vulnerability

If you discover a security issue, please do not publish exploit details immediately.

Preferred process:

1. Open a GitHub issue with the title prefix `[security]` and share minimal reproduction details.
2. If the issue contains sensitive details, ask for a private contact channel in that issue.
3. Allow time for a fix before public disclosure.

## Response expectations

- Initial triage target: within 7 days.
- Fix or mitigation target: depends on severity and reproducibility.

## Scope

Examples in scope:

- Code execution or privilege escalation through extension logic.
- Data exfiltration caused by extension behavior.
- Manifest or permission misconfigurations with security impact.

Out of scope:

- Issues in third-party translation services.
- Browser-specific bugs unrelated to this codebase.
